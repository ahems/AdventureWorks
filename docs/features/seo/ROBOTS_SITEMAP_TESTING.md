# Robots.txt and Sitemap.xml Testing Guide

## Overview

The `test-robots-sitemap.sh` script validates that the AdventureWorks application properly implements SEO standards for search engine crawlers. It tests the robots.txt file, sitemap.xml generation, and URL accessibility.

## What It Tests

### 1. Robots.txt Validation

- ✅ File accessibility (HTTP 200)
- ✅ Contains required `User-agent` directives
- ✅ Contains `Allow` directives for crawlable paths
- ✅ Contains `Sitemap` directive pointing to sitemap.xml
- ✅ Contains `Crawl-delay` for politeness

### 2. Sitemap.xml Structure

- ✅ Proper XML declaration
- ✅ Correct sitemap namespace (`http://www.sitemaps.org/schemas/sitemap/0.9`)
- ✅ Valid `<urlset>` root element
- ✅ All URLs have `<loc>` elements
- ✅ Contains `<lastmod>` for content freshness
- ✅ Contains `<priority>` for page importance
- ✅ Contains `<changefreq>` for update frequency

### 3. Content Coverage

- Product pages (all finished goods)
- Category pages
- Static pages (homepage, products, sale, FAQ, contact)

### 4. URL Accessibility

Tests a sample of URLs from the sitemap to ensure they're reachable.

## Usage

### Run the Test

```bash
./test-robots-sitemap.sh
```

### Prerequisites

- Azure infrastructure must be deployed (`azd up`)
- Environment variables must be set (automatic after azd deployment)

### What You'll See

The test outputs colored results:

- 🟢 **Green checkmarks**: Tests passed
- 🔴 **Red X marks**: Tests failed
- 🟡 **Yellow info**: Informational messages

### Test Results

Results are saved to `test-results/robots-sitemap-test-<timestamp>.txt` with:

- Test configuration (URLs tested)
- Validation results for robots.txt
- Validation results for sitemap.xml
- Content coverage statistics
- Sample URL test results
- Overall SEO compliance status

## Architecture

### Robots.txt Location

- **File**: `app/public/robots.txt`
- **Served by**: Azure Static Web Apps
- **URL**: `https://<app-url>/robots.txt`

### Sitemap.xml Generation

- **Function**: `api-functions/Functions/SitemapFunction.cs`
- **Endpoint**: `/api/sitemap.xml`
- **Hosted by**: Azure Container Apps (Functions)
- **Dynamic**: Generated on-demand from Azure SQL database

### Redirect Flow

1. Static Web App serves `/robots.txt` directly
2. Robots.txt references `/sitemap.xml` on the app domain
3. Static Web App config redirects to Functions endpoint
4. Functions dynamically generate sitemap from database

## Integration with Search Engines

### How Search Engines Use This

1. **Crawler discovers robots.txt**

   ```
   GET https://your-app.azurestaticapps.net/robots.txt
   ```

2. **Robots.txt tells crawler about sitemap**

   ```
   Sitemap: https://your-app.azurestaticapps.net/sitemap.xml
   ```

3. **Crawler fetches sitemap (redirected to Functions)**

   ```
   GET https://your-app.azurestaticapps.net/sitemap.xml
   → 301 https://functions.azurecontainerapps.io/api/sitemap.xml
   ```

4. **Sitemap provides all URLs to crawl**
   ```xml
   <url>
     <loc>https://your-app.azurestaticapps.net/product/123</loc>
     <lastmod>2026-01-27</lastmod>
     <priority>0.8</priority>
     <changefreq>weekly</changefreq>
   </url>
   ```

## Common Issues

### Sitemap URLs Return 404

**Problem**: Static pages in sitemap use the Functions URL instead of the app URL.

**Why**: The sitemap is generated by the Functions app which doesn't host the frontend.

**Solution**: This is expected for static pages. Product URLs should work when navigated from the actual app domain.

### Redirect Required for Sitemap

**Problem**: Accessing `/sitemap.xml` returns HTTP 301.

**Why**: Static Web Apps redirects to the Functions endpoint via `staticwebapp.config.json`.

**Solution**: This is intentional. Search engines follow redirects. The test handles this with `-L` flag.

### Missing Environment Variables

**Problem**: `API_FUNCTIONS_URL not found in azd environment`

**Solution**:

```bash
azd up  # Deploy the infrastructure
azd env get-values  # Verify variables are set
```

## Customization

### Modify robots.txt

Edit `app/public/robots.txt`:

```
User-agent: *
Allow: /
Disallow: /admin  # Block admin pages

Sitemap: https://your-app.net/sitemap.xml
```

### Modify Sitemap Content

Edit `api-functions/Functions/SitemapFunction.cs`:

- Add/remove static pages in `AddStaticPagesAsync()`
- Modify priority/changefreq values
- Filter products (e.g., only show in-stock items)

### Test Different Environments

```bash
# Override URLs for local testing
FUNCTION_URL="http://localhost:7071" \
APP_URL="http://localhost:3000" \
./test-robots-sitemap.sh
```

## Best Practices

1. **Run after deployments** - Verify sitemap updates after product catalog changes
2. **Monitor URL counts** - Significant changes may indicate data issues
3. **Check HTTP codes** - All URLs should return 200 or 3xx redirects
4. **Update robots.txt sitemap URL** - Should match your production domain
5. **Test with real crawlers** - Use Google Search Console to verify

## Troubleshooting

### Test hangs on URL checks

**Cause**: Network timeouts or slow responses

**Solution**: Reduce sample size or add timeout to curl:

```bash
curl -s -L --max-time 5 -o /dev/null -w "%{http_code}" "$url"
```

### Sitemap shows wrong base URL

**Cause**: Function derives URL from HTTP request headers

**Solution**: Check `X-Forwarded-Host` headers in Container Apps ingress config

### Test can't find azd environment

**Cause**: Running outside dev container or without azd login

**Solution**:

```bash
azd auth login
azd env refresh
```
